Web Ethics

Review

It is improper to:

  • link to images, scripts, etc. not hosted on your own server
  • use the code or design of another site as your own
  • use copyrighted photos, graphics, multimedia elements, etc. without permission
  • use proprietary source code

Understanding open source

  • proprietary, closed-source code is a "wasteful distribution of resources"
  • common goal: support opening, modification, and redistribution of code and software
  • anyone can contribute
  • introduced the "copyleft" principle
    • using copyright law to ensure openness

Open-source history

  • BSD UNIX forked from Bell Labs' version
    • Popular among research laboratories, universities
  • DARPA funds BSD to provide a version of UNIX with networking capabilities for ARPANET
  • AT&T's licensing fee reaches $200,000
  • BSD lives on in several operating systems today (i.e. Mac OS X)
  • Linux also a response to AT&T's decision to make UNIX proprietary

Open-source software

  • Operating systems: Linux, Chromium OS
  • Languages: Ruby, PHP, Python
  • Browsers: Firefox, Google Chrome (based on Chromium)
  • Databases: MySQL, MongoDB
  • Applications: Apache (web server), OpenOffice (suite)

Open-source web software

  • Bootstrap
  • Foundation
  • Blueprint CSS
  • Font Awesome
  • jQuery

Shared with ♥

  • Many creators will allow users to link to the code base of an open-source project on a remote server
  • CDN: Content Delivery Network
    • hosted on a server optimized for speed, sharing
    • using CDNs is acceptable, encouraged

CDN examples

  • Bootstrap CDN
  • jQuery CDN

Creative Commons

https://creativecommons.org/

  • an extension of "copyleft"
  • allows a creator to retain ownership
  • grants rights to users to copy, distribute, edit, remix, build upon

Creative Commons Licenses

https://creativecommons.org/licenses/

  • Attribution
  • Attribution-ShareAlike
  • Attribution-NoDerivs
  • Attribution-NonCommercial
  • Attribution-NonCommercial-ShareAlike
  • Attribution-NonCommercial-NoDerivatives

Public Domain

  • when intellectual property rights have expired (or are waived)
  • laws vary by country
  • works of the U.S. government are automatically released into public domain
    • work prepared by an "officer or employee" of the federal government
    • some items (i.e. logos) may still be protected by trademark laws

Summary

  • It is not okay to take and use anything off the web for your own purposes without permission
  • All work should be your own, utilize your own resources
  • It is okay to use:
    • open-source code
    • public domain and Creative Commons works
  • Always provide attribution (whether requested or not)
