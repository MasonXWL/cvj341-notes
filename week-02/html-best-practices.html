HTML Best Practices

Use the proper DOCTYPE

A DOCTYPE tells the browser which version of HTML you are using.

There are several DOCTYPES. We will be using the one which tells browsers our code is written in HTML5.

<!DOCTYPE html>

Declare a character set

A character set tells the browser which characters we want our content to be encoded in.

There are many character sets beyond the ASCII noted when discussing text editors. These sets include different kinds of glyphs and foreign characters.

UTF-8 is recognized as a good standard in web design because it includes more than 1 million characters.

<meta charset="UTF-8" />

Close your tags

Good: <p>Text</p>

Bad: <p>Text

There are a few exceptions which are called self-closing tags.

  • <br />
  • <hr />
  • <img />

Write elements, attributes and values in lowercase

HTML is not case sensitive, but this is an accepted best practice to write elements, attributes and values in lowercase for backward compatibility with XHTML.

<p class="intro">This is a sentence.</p>

Use quotes around attribute values

Using quotes around attribute values adds visual cues for easier-to-read code. Quotes also help avoid bugs when working with CSS and Javascript.

<img alt="Sebastian the Ibis" class="photo" src="photo.jpg" />

All attributes should have values

Most attributes require a value. Some do not.

Adding a value, even when not required, will help when working with CSS and Javascript.

<hr noshade="noshade" />

Do not use double dashes in comments

<!-- This is a comment -->

<!-- If you must use double dashes, use two single dashes separated with a space - - like so. -->

Because double dashes are used at the beginning and end of a comment, using them within a comment may invalidate your code or create a bug.

Encode special characters

Special characters, or HTML entities, begin with an ampersand (&) and end with a semicolon (;). These may be glyphs, or characters used in foreign languages.

Nice entity cheat sheets include:

  • W3C: HTML5 Character Entities
  • Character Code
  • Entity Code

Examples

  • Double dagger: ‡
  • Spade: ♠
  • Pound Sterling: £
  • Euro: €
  • Copyright: ©

Many characters have an entity name and entity number. Most modern browsers will recognize entity names. If the symbol appears broken, however, try using the entity number instead.

Indent code

HTML is not sensitive to white space.

Indent code by two spaces for each level. This makes it easier to match block-level opening and closing tags, making your code easier to read and maintain.

Most code editors can be set to adjust the indent spacing to utilize the tab key. Make sure it is set to indent with two spaces and not a tab (which is five spaces).

The amount of spacing you use between lines is a matter of preference. Find a pattern which suits you and remain consistent.

By keeping your code clean and organized, others who may also work on the same code set will find it easier to work with.
