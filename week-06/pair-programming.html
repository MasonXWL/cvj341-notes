Pair Programming

What is pair programming?

  • Collaboration
  • Communication
  • Learning
  • Productive
  • Efficient

How to pair

  • Find someone to code with
  • One person takes the role of "driver"
    • Responsible for inputting data
  • One takes the role of "navigator"
    • Responsible for reviewing code

Getting started

  • Define a task
  • Aim for one goal at a time
  • Driver
    • Complete the task, with your navigator as a safety net
  • Navigator
    • Communicate positives and negative
  • Always share your thought process with the other person

Switching roles

  • Try each role for 30 minutes at a time
  • Try working with others
  • Challenge: Can you pair without communicating? :)
