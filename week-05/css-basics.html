CSS Basics

What is CSS?

  • Cascading Style Sheets
  • Describe how an HTML document should be presented
  • Predominantly visual
    • colors, fonts, layout, animation, etc.
  • We can specify aural styles
    • volume, pauses, cues, etc.
  • And Braille, too!

CSS History

  • Developed by Håkon Wium Lee and Bert Bos in 1994
  • Improved upon the capabilities of the HTML 1.0 and the visually minded Mosaic
  • CSS1 adopted by Netscape, Internet Explorer in 1996
  • CSS2 introduced in 1998
    • But it was very buggy, so W3C created a working draft of CSS2.1 in 2005
  • Today: CSS3, a work in progress

Why Cascading?

  • Styles applied to a parent element are, by default, inherited by child elements
  • But there's a hierarchy as to which rules take hold:
    • The last rule always wins out
    • A more specific rule will trump a less specific one
    • !important override

Ways to use CSS

Inline Styles

We can use the style attribute inside elements:

<p style="color: red;">This is red text.</p>

DO NOT DO THIS. BAD! BAD! BAD!

Internal Styles

We can include a <style> tag inside the head of each HTML document.

<style>
  p { color: red; }
<style>

Not so bad, but not very good either.

External Style Sheets

We can create a .css document which we can link to in our HTML document.

<link href="css/style.css" rel="stylesheet" />

You may see the attribute/value of type="text/css" in some HTML documents. This is optional. Browsers defer to CSS as the default stylesheet language.

No tags are used in a .css document, just CSS blocks.

Why are external stylesheets preferred?

You maintain a web site of 100,000 pages. You decide to include:

  • HTML presentational elements such as <b> or <hr/>
  • Inline or internal styles which set your paragraphs in Helvetica and green.

I hate Helvetica! I hate green! — your boss >:(

You: :'-(

You now have to search through 100,000 pages to find these elements or styles and remove them.

But there has to be a better way, right?

What if you had used an external style sheet instead?

<link href="css/style.css" rel="stylesheet" />

We can fix the problem with four lines of code.

p {
  font-family: "Arial", sans-serif;
  color: black;
}

You, now: :-)
