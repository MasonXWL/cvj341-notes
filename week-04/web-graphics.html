Web Graphics

Types of graphics

Raster

  • a grid of pixels
  • cannot scale up
  • edited using Photoshop

Vector

  • points, lines, curves, shapes
  • can be scaled up
  • edited using Illustrator

Compression

  • allows for smaller file sizes
  • necessary for images, multimedia
  • data encoding disregards some information
  • compression » loss of quality » smaller file size

Lossy vs. lossless compression

Lossy

  • file loses data in compression
  • smaller file sizes, but quality is degraded

Lossless

  • file does not lose data in compression
  • massive file sizes, but quality is retained

Web graphic formats

GIF: Graphics Interchange Format (.gif)

  • limited to a palette of 256 colors
  • good for low-color graphics such as logos
  • small animations
  • transparency

PNG: Portable Network Graphics (.png)

  • a replacement for GIFs (PNG-8)
  • large color palette (PNG-24)
  • transparency
  • smoother (anti-aliased) graphics
  • no animation capabilities

JPEG: Joint Photographic Experts Group (.jpg or .jpeg)

  • millions (billions?) of colors
  • used for photographic images
  • compression may leave 'artifacts'
  • progressive display reveals a portion of the image over time

SVG: Scalable Vector Graphics (.svg)

  • vector format for the web
  • behaviors described in XML
  • supported in all major browsers since 2011
  • ...but not yet fully adopted

Web graphics toolkit

Adobe Photoshop

  • used to edit raster images
  • File » Save for Web
  • allows you to save in raster formats (.gif, .jpg, .png)

Adobe Illustrator

  • used to edit vector images
  • File » Save for Web
    • allows you to save in raster formats (.gif, .jpg, .png)
  • File » Save As...
    • allows you to save in vector format (.svg)

Preparing web graphics

  • always retain the original image
  • utilize Photoshop layers
    • Layer » Duplicate Layer
  • save all work as a .pdf (Photoshop) or .ai (Illustrator)
  • export to appropriate web format
    • once an image is compressed, data is lost
  • work in RGB color mode
    • CMYK color mode is for print
  • raster images for web should be 72 dpi*
    • Image » Image Size...
    • print requires 300 dpi images
    • * may require modification for retina displays
  • never re-size images by increasing height, width
  • do not distort images with unbalanced proportions
    • automatically taken care of in Photoshop
    • make sure "constrain proportion" icon is selected
  • photos should always be saved as .jpg
  • graphics should always be saved as .gif or .png
    • .png is the preferred format
  • consider size vs. quality when saving
    • Does your design focus on strong imagery?
    • What considerations are you making for mobile?

Practice

Included in the web-graphics repo is an img directory containing two high-resolution images. Download these and experiment with different settings in the Save for Web panel of Photoshop. See how adjusting the file format and quality impacts the image, as well as the file size.

Determining the right file size vs. quality is often a judgement call. Find what you believe is a fair balance.
